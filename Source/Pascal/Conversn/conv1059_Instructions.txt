**NOTE: CONV1059.SQL WILL STOP OTHER PROGRAMS  **
-------------------------------------------------
CONV1059.SQL will stop other programs from being able to use the Pervasive database engine whilst it is running.
The FORMIX database should be tidied as much as possible before applying this update.
Use FORMIX menu options ‘Order Cleardown’ and ‘Ingredient Usage Cleardown’ to do this.



1. Save contents of supplied update file into FORMIX\UPDATE folder.
2. Stop FORMIX program running on workstations.
3. Stop SFORMIX program running on terminals (enter user name 'EXIT' at login prompt).


4. From a command prompt in the FORMIX folder run (or do equivalent from Windows Explorer):-
  RENAME FORMIX.EXE *.058


5. From a command prompt in the FORMIX\FIL folder run (or do equivalent from Windows Explorer)::-
  COPY FXCOST.FIL *.058
  COPY FXNET.FIL *.058
  COPY FXTRANS.FIL *.058

  ..\UDPATE\CONV1059      (renames original LOT.FIL to LOT.058)
  COPY *.DDF *.BAK
  RENAME LOT.FIL *.NF

6. From Pervasive Control Centre on the server execute the following on the FORMIX database:-

  NOTE: if the 'LotIRef' table does not exist in the FORMIX database remove the DROP TABLE "LotIRef" statement from conv1059.SQL file.

  Execute the entire contents of the SQL file: conv1059.SQL


7. From a command prompt in the FORMIX\FIL folder run (or do equivalent from Windows Explorer)::-
  RENAME LOT.NF *.FIL


8. From a command prompt in the FORMIX folder run (or do equivalent from Windows Explorer)::-
  COPY ..\UPDATE\FORMIX.EXE


9. Restart FORMIX program on a workstation.
